A Gamemaker's Guide to Style

All you need to develop CSS in your game.

A simple introduction to CSS for all of you Gamemakers trying to get your game looking exactly like you are imagining.

The Basics: Preparation and Assumptions

This guide assumes you are on desktop (I only own a Windows machine, you can contact me on GitHub for info on other OS's).

I also assume you have read Orteil's guide on CSS, while you don't need to read the 'Read more about CSS' link, which sends to a website whose purpose is out of IGM and can only help if you know things like HTML.

If you don't have a preferred text editor, or if yours doesn't support CSS (and SASS, later) syntax highliting, I suggest installing Visual Studio Code.

If you already know some HTML, you can skip parts in gray.

Selectors

Create a new file (in VSCode, File > New File) anc all it (or save it as) style.css

When styling stuff, you can refer to elements, classes, IDs, pseudoclasses and pseudoelements.

In order:

  • Elements are thinks like 'all links' or 'all images', or all <p>s
  • Classes are thinks that can identify multiple elements whihc share a property, like 'all boxes' when styling IGM.
  • Ids are identifiers given to a single item in the document, like 'the building box'
  • Pseudoclasses and pseudoelements are advanced topics and I'll discuss them later

Elements aren't really needed in IGM, where the most important selectors are classes and IDs.

When referring to a class, you prefix the name of said class with ., so for example 

.foo {
  
}

refers to class 'foo'. For IDs, you prefix with #. 

#foo {
  
}

You can also combine selectors, so if I want to target an element with id foo only if it has the class bar (because you toggle it with a setting), you can write 

#foo.bar {
  
}

which is different from 

#foo .bar {
  
}

which means 'all elements with class bar which are children of the element with ID foo'.

In short, when there is no space you are requesting an element (or more than one) with multiple properties, while when there are spaces you are checking for parents. For example, 

#foo.bar .baz {
  
}

targets all elements with class baz if they are children of elements with ID foo and class bar.

Properties

Once you are inside the {curly braces} of a selector, you want to write some properties that define the style of the element you are targeting.

There are a lot of different properties and explaining them all is not my job.

Starting from this step, we are going to use a demo HTML file to test our CSS before switching to a game.

Create a new file in the same folder as style.css and name it index.html. Inside, write this code: 

<!DOCTYPE html> <!-- This line indicates that our file is HTML -->
<html> <!-- This is the root of our document -->

<head> <!-- In the head we define things we (mostly) cannot see -->
  <meta charset="utf-8"> <!-- This means we are using the UTF-8 charset -->
  <link rel="stylesheet" href="style.css" /> <!-- This includes our CSS file -->
</head>

<body> <!-- Here are the things we can see -->

</body>

</html>

<!-- and --> indicate comments.

We are only going to edit the <body>. Open this file in your favorite browser.

You should see an empty page.

Editing fonts

One of the first things you might like to do is change the font. If we add something (like Hello!) in our body, we can see it on the browser with the default font (probably a serif font, but some browsers allow changing it via settings). That sucks!

Hello!

To change the font, we can go in our CSS and write: 

body {
  font-family: /* Whatever font you like */ 'Arial';
}

We are introducing a few things here:

  • Properties
  • Values
  • Syntax

Properties are like font-family. They are (mostly) fixed strings, and depending on the string different styles are impacted.

Values can be numbers, colors, strings... In this simple case we are using a string, indicating we want Arial as the font.

Properties and values are separated by colons, and each property-value pair is separated by a semicolon.

/* and */ delimit comments.

Hello!

Other font and text properties

We can edit colors, weight, decorations and much more.

Let's edit the body of our HTML test file 

<div class="italic"> <!-- This is a container for elements -->
  Hello!
</div>
<div class="bold">
  Hi!
</div>
<div class="red">
  Hullo!
</div>
<div class="limeish">
  I'm out of greetings!
</div>

In our CSS, we can write some (utility) classes. 

.italic {
  font-style: italic; /* Italic is a keyword */
}
.bold {
  font-weight: 700; /* A number */
}
.red {
  color: red; /* A color keyword */
}
.limeish {
  color: #00e347 /* A color */
}
